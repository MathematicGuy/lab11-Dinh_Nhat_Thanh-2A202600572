# Mission: Guardrails, HITL, and Responsible AI for Day 11

## Why
I want to understand guardrails and human-in-the-loop design well enough to explain them clearly and implement them correctly in this Day-11 banking-agent project. The practical outcome is being able to decide which safety checks belong in input rails, LLM rails, output rails, and HITL routing without blindly adding latency or weakening safety.

## Success looks like
- I can explain defense-in-depth for agent safety using concrete Day-11 repo examples.
- I can map a red-team user story to the right layer: input rail, LLM rail, output rail, or HITL.
- I can justify tradeoffs between stricter safety and lower latency for a banking assistant.

## Constraints
- Teaching should stay grounded in this repository's Day-11 banking-agent lab and assignment.
- I want implementation-oriented lessons, not abstract AI ethics only.
- I want to learn incrementally through short lessons and reusable references.

## Out of scope
- Deep model training or alignment research
- Broad AI policy debates not directly useful for this project
- Framework-specific features that do not affect the Day-11 implementation decisions
